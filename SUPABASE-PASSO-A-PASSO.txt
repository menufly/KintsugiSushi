PASSO A PASSO - KINTSUGI SUSHI COM SUPABASE

1. Crie uma conta em https://supabase.com

2. Clique em New project.

3. Depois que o projeto abrir, vá em SQL Editor.

4. Abra este arquivo no seu computador:
   supabase-setup.sql

5. Copie todo o conteúdo do supabase-setup.sql, cole no SQL Editor do Supabase e clique em Run.

6. No Supabase, vá em Project Settings > API.

7. Copie:
   - Project URL
   - anon public key

8. Abra este arquivo:
   config.js

9. Preencha assim:

   window.KINTSUGI_SUPABASE = {
     url: "COLE_A_PROJECT_URL_AQUI",
     anonKey: "COLE_A_ANON_PUBLIC_KEY_AQUI",
     table: "pedidos"
   };

10. Publique os arquivos da pasta publicar-netlify no GitHub.

11. Se for usar GitHub Pages:
   - entre no repositorio no GitHub
   - Settings
   - Pages
   - Source: Deploy from a branch
   - Branch: main
   - Folder: /root
   - Save

12. Teste:
   - faca um pedido no site
   - abra admin.html
   - login: administracao
   - senha: 123456
   - altere o status
   - volte no pedido e aguarde atualizar

13. Para login com e-mail e senha:
   - no Supabase, va em Authentication
   - va em Providers
   - confira se Email esta ativado
   - para testes, voce pode desligar a confirmacao de e-mail se quiser entrar imediatamente

14. Para login com Google:
   - no Supabase, va em Authentication > Providers
   - ative Google
   - siga as instrucoes do Supabase para preencher Client ID e Client Secret do Google
   - em Authentication > URL Configuration, coloque o link do seu site em Site URL
   - em Redirect URLs, adicione o link do seu site do GitHub Pages

15. Depois de atualizar o SQL:
   - rode novamente o conteudo de supabase-setup.sql no SQL Editor
   - envie index.html e supabase-setup.sql atualizados para o GitHub

OBSERVACAO IMPORTANTE
O cliente agora usa autenticacao do Supabase. A administracao ainda usa login simples no HTML para facilitar seus testes.
